## なぜ「人気順」では不十分か

最初の調査リストは登録者数ランキングになっていた。
それでは「バズレシピ・節約・韓国料理」に偏ったフィードができあがるだけで、
Uchimiseのコンセプト実現にはならない。

**問い直すべき軸：**

> 「行きつけの食料品店の物知りな店主」は、何を棚に並べるか。

- 知名度が高いから並べるのではなく、**ユーザーが知らなかったがゆえに発見になる**から並べる
- 今日の旬・今週使い切りたい野菜・普段は素通りする食材——そういう視点での選書
- 「5種類の発見パターン」で棚を設計し、それを補うチャンネル/アカウントを選ぶ

## 発見フィード 5つのパターン

```
Pattern A  旬と素材の物語
           ────────────────────────────────────────────
           「今の季節ならではの食材に、気づかせてくれる」
           ユーザーが知らなかった食材や調理法との出会い。
           スーパーで素通りしがちな食材が主役になる瞬間。

Pattern B  隣国・隣文化の食卓
           ────────────────────────────────────────────
           「知ってるようで知らない、台湾・ベトナム・
            タイ・中東・スペインの家庭料理」
           外食でしか食べていないものが「家でも作れる」と知ること。

Pattern C  技術と知識の一段深み
           ────────────────────────────────────────────
           「なぜこの手順なのか、がわかる」
           料理が「なんとなく」から「ちゃんと理解している」に変わる体験。
           出汁・発酵・スパイス・火入れの基礎。

Pattern D  平日の現実に寄り添う再現性
           ────────────────────────────────────────────
           「疲れた帰宅後でも、ちゃんと食べたい」
           冷蔵庫ドリブン・時短・少量・翌日弁当への転用が利く。

Pattern E  ちょっと特別な日の一品
           ────────────────────────────────────────────
           「誰かに作る日、自分にご褒美の日」
           レストランではなく「家で」「手作りで」が成立する上限ライン。
```

## 意図的に外したカテゴリ・理由

| カテゴリ | 判断理由 |
|---|---|
| **ダイエット・カロリー管理専門** | 義務感・自己管理プレッシャーを生む。「毎日来なくていい店」コンセプトと相反 |
| **ASMR・食べるだけ系** | レシピ抽出ができない。発見フィードの機能要件を満たさない |
| **チートデイ・大盛り系** | 一人暮らし社会人ペルソナのリアルな献立ニーズと乖離 |
| **料理コンテスト・プロ競技系** | 再現難易度が高く「作る気持ち」につながらない |
| **チャンネル登録者数500万超の最大手のみ** | 既にユーザーが知っている＝発見体験にならない |

## 多国籍ジャンル補強（追記）

> 「自分だったらメキシコ大使館チャンネルも見る」という視点から追加調査。
> 人気クリエーターではなく「ソースの種類」を広げる方向で整理する。

### なぜ大使館・公的機関チャンネルか

個人インフルエンサーが「家庭料理っぽく作ってみた」ではなく、
**その国の政府・文化機関が「本国の本物の食卓」として発信している**点が価値。
発見フィードの中で、これらは「物知りな店主が仕入れてきた話」の最たる例になる。

| ソースタイプ | 一般クリエーターとの違い |
|---|---|
| 大使館公式 | 「外食の異国料理」ではなく「その国の家庭の普通の夕食」が見える |
| 国際放送・文化省系 | 政府認定のレシピ精度・食文化背景の説明がつく |
| 在日ディアスポラ | 「日本の食材でどう作るか」という翻訳作業が込められている |

### 多国籍フィードのソース選定指針（更新版）

大使館・政府系チャンネルを使う際の注意点：

- **更新頻度が低い**ことが多い → フィード鮮度のために類似ジャンルの個人チャンネルと組み合わせる
- **字幕・テキスト情報が少ない動画もある** → Phase 1の抽出精度が下がる可能性。URLのみ保存の「Private投稿」扱いに近いフォールバック設計が必要
- **「この国の料理」という文脈情報がコンテンツの価値** → レシピカードに国旗・食文化の一言メモを表示するUIが、他のチャンネルより効果的に機能する

## 空白ジャンルの今後の候補調査

現時点でまだ適切なソースが見つかっていない領域。
次のフェーズで継続調査：

- **沖縄・東北・北陸の郷土料理** チャンネル
- **和菓子・煎餅・饅頭を手作りする** チャンネル（旬果物を使う和菓子）
- **ベトナム家庭料理（日本在住ベトナム人運営）** チャンネル ← ハングリーハスキーで部分補完可
- **フランス家庭料理・ビストロ惣菜** 日本語解説チャンネル ← BSフジ大使館の食卓で部分補完可
- **旬の魚専門** （捌き方から料理まで）
- **中南米系ディアスポラ（ペルー・コロンビア）** チャンネル

## 実装への示唆

発見フィードを「人気コンテンツの集約」として設計すると、
ユーザーはやがて「自分でYouTubeを検索するのと変わらない」と感じる。

Uchimiseの発見タブが他と違うのは：

1. **棚の選書に意志がある**（なぜこれが今日のあなたにか、が説明できる）
2. **ジャンルの偏りが発見の障害になる**（韓国料理・時短に偏ると「知ってるもの」しかない）
3. **小さなクリエーターにも光が当たる**（128万人のにぎりっ娘よりも5000人の台湾料理の人が、あなたにとっての発見になることがある）

pgvectorの類似検索を「人気度」だけでなく「パターンの多様性確保」に使うことで、
フィードが単調なレコメンデーションループに陥るのを防ぐ設計が必要。
