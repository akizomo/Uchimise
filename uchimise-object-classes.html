bookmark_statusenum必須🆕 saved / want_to_cook / cooked

状態遷移 (phase)
phase1
Phase 1 完了
タイトル・サムネイル・材料品名のみ。分量・手順は未確定。Goldenrod バッジ表示
phase2
Phase 2 完了（LLM正規化済み）
分量・手順・タグが正規化済み。Goldenrod バッジ消える
manual
手動入力
非対応URL or 字幕なし。ユーザーが手動で入力したレシピ
URL保存 → phase1（即時）
→ BullMQ → phase2（30秒以内）
字幕なし → manual_required → manual
状態遷移 (bookmark_status)
saved
保存済み（まだ調理予定なし）
want_to_cook
今度作りたい（献立候補）
cooked
作り済み（CookingRecord と連動）

extraction_jobs
Phase 1/2 の抽出ジョブ管理。リトライ・エラー履歴の追跡
🆕 新規MVP
プロパティ
フィールド型必須説明
iduuid PK必須
recipe_iduuid FK必須recipes.id
phaseenum必須phase1 / phase2
statusenum必須pending / processing / done / failed
error_codetext任意private_post / no_caption / quota_exceeded / llm_error
attemptsint必須試行回数。default 0
started_attimestamptz任意処理開始時刻
completed_attimestamptz任意完了時刻
状態遷移
pending
キュー待ち
processing
BullMQ ワーカー処理中
done
正常完了。recipes テーブル更新済み
failed
エラー。error_code に理由記録
pending → processing → done
↓ 失敗
failed（attempts += 1, max 3回）
3回失敗 → recipe は phase1 のまま保存
UI対応 / 関係
SNS保存フローPhase 2 バナー
status=processing 時に「材料を精密化中...」バナー表示。Supabase Realtime で status 監視
